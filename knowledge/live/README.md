# Live — bitácora viva

Diario de trabajo del proyecto. Trazabilidad narrativa de qué se hizo y por qué.

- `daily/` — entradas por fecha.

## Regla de oro

Lo que se estabiliza (una decisión de arquitectura, una definición madura) se
promueve a `decisions/` (ADR) o a `current_state.md`. La bitácora es el borrador
vivo; las decisiones y el estado son el registro consolidado.
